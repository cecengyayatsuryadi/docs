# Session Start Checklist

> Run this checklist at the **beginning** of every session.
> This is how you load context when you can't remember the last session.
> **Skipping this = working blind. You will repeat mistakes.**

**Last updated:** YYYY-MM-DD

---

## Checklist

### 1. Load Context (Start Here)
- [ ] Read [current-state.md](current-state.md):
  - **Handoff Notes** — what did the last session leave for you?
  - **WIP Context** — is there interrupted work to resume?
  - **Project Status** — what's the health and current work?
- [ ] Read [board.md](board.md) — what's the current sprint and priorities?
- [ ] Read [repo-working-agreement.md](repo-working-agreement.md) — what are the rules?

### 2. Understand the Task
- [ ] Read project-specific docs (README, PRD, specs) if they exist
- [ ] Check [operations/decision-log.md](operations/decision-log.md) for prior decisions related to your task
- [ ] Check [operations/lessons-learned.md](operations/lessons-learned.md) for relevant past lessons

### 3. Check for Hazards
- [ ] Check [operations/risk-register.md](operations/risk-register.md) for known risks
- [ ] Check [operations/action-items-tracker.md](operations/action-items-tracker.md) for unresolved items
- [ ] Check [git-management-policy.md](git-management-policy.md) if you'll be making commits
- [ ] Check [agent-working-agreement.md](agent-working-agreement.md) for forbidden actions and escalation triggers

### 4. Set Up Environment
- [ ] Verify you're on the correct branch
- [ ] Verify dependencies are installed
- [ ] Verify the project builds cleanly before making changes
- [ ] Verify tests pass before making changes (so you know what's pre-existing)

## Quick Start (Minimum — 2 Minutes)

If pressed for time, at minimum read:

1. [current-state.md](current-state.md) → **Handoff Notes section only**
2. [board.md](board.md) → **Active Tasks only**

## First Session in a New Repo?

If this is your first time in this repo:

1. Read [README.md](README.md) for system overview
2. Read [setup-guide.md](setup-guide.md) if the docs haven't been configured yet
3. Read [docs-map.md](docs-map.md) to understand where everything lives
4. Then follow the normal checklist above
