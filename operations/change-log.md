# Change Log / Release Notes

> User-facing changes and releases. Not for internal refactors or documentation changes.

**Last updated:** YYYY-MM-DD

---

## When to Add an Entry

- A feature is shipped to users
- A bug fix affects user behavior
- A breaking change is deployed
- A performance improvement is measurable

---

## Format

Follows [Keep a Changelog](https://keepachangelog.com/) conventions:
- **Added** — new features
- **Changed** — changes to existing functionality
- **Deprecated** — features that will be removed
- **Removed** — features removed
- **Fixed** — bug fixes
- **Security** — vulnerability fixes
